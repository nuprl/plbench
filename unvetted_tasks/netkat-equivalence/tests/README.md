# Equivalence fixture audit guide

The OCaml verifier discovers every `.nk` file in `cases` and compares the
reported results with its same-stem `.expected` file. Each fixture must name
its theme and source and must document every property. Expected results are
manually audited data, not outputs generated by a reference implementation.

The 15 fixtures contain 96 independently scored equations: 64 equivalent and
32 inequivalent. The suite is organized around:

- the KAT and NetKAT axioms presented in the POPL 2015 paper;
- examples where endpoint relations agree but packet histories differ;
- stars containing `dup`, including unbounded and cyclic history languages;
- distributivity, Boolean tests, assignments, and full-domain complements;
- path-sensitive load balancing, topology, and translation-validation shapes;
- long distinguishing traces intended to reject fixed-depth execution checks.

Positive and negative equations are interleaved. Test discovery depends only
on the contents of `cases`; this README is an audit index, not a verifier list.
