# History-sensitive NetKAT equivalence

## Command line

Your program must be executable as:

```text
/app/netkat-equivalence INPUT.nk
```

For a valid input, print one line for every `check`, in source order:

```text
CHECK_NAME: equivalent
CHECK_NAME: not equivalent
```

Print no other text to standard output and exit with status 0. Diagnostics may
go to standard error. For an invalid input, exit nonzero. Grading inputs are
valid.

## Headers and values

Every packet has exactly four headers:

```text
src_ip    source IPv4 address
dst_ip    destination IPv4 address
src_port  source transport port
dst_port  destination transport port
```

An IPv4 value uses canonical dotted-decimal notation with four octets from 0
through 255. An octet has no leading zero unless it is zero. A port is a
decimal integer from 0 through 65535, without leading zeros unless it is zero.

The packet space is the full Cartesian product

```text
IPv4 × IPv4 × Port16 × Port16
```

and therefore contains `2^96` packets. Inputs do not declare smaller domains.

## Lexical syntax and grammar

Whitespace is insignificant. A `#` begins a comment extending to the end of
the line. Check names match `[A-Za-z_][A-Za-z0-9_-]*`. The words and header
names in the grammar are reserved.

In this grammar, `{ x }` means zero or more repetitions and alternatives are
separated by `|`. Operators are listed from low to high precedence.

```text
program     ::= check { check }
check       ::= "check" IDENT ":" policy "==" policy ";"

policy      ::= sequence { "+" sequence }
sequence    ::= repetition { "." repetition }
repetition  ::= policy_atom { "*" }
policy_atom ::= "drop"
              | "id"
              | "dup"
              | "filter" "(" predicate ")"
              | header "<-" value
              | "(" policy ")"

predicate   ::= conjunction { "|" conjunction }
conjunction ::= negation { "&" negation }
negation    ::= "!" negation | predicate_atom
predicate_atom
            ::= "true"
              | "false"
              | header "=" value
              | "(" predicate ")"

header      ::= "src_ip" | "dst_ip" | "src_port" | "dst_port"
value       ::= IPV4 | PORT
```

Thus `*` binds tightest in policies, then `.`, then `+`. Predicate precedence
is `!`, then `&`, then `|`. Binary operators associate to the left, although
their denotations are associative. An IP header may only use an IPv4 value and
a port header may only use a port value. A file contains at least one check,
and check names are distinct.

## Packet histories

A packet history is a nonempty sequence of packets:

```text
<pk_current, pk_previous, ..., pk_initial>
```

The leftmost packet is the current packet. Policies only inspect or modify the
current packet, except that `dup` records a new history entry. A policy denotes
a function from an input history to a set of output histories.

## Predicate semantics

- `true` accepts every history and `false` accepts none.
- `h = v` accepts a history exactly when the current packet's header `h` is
  `v`.
- `!a`, `a & b`, and `a | b` denote complement, intersection, and union of
  tests relative to the full packet space. A successful test returns its input
  history unchanged; a failed test returns no history.

## Policy semantics

For an input history `<pk, rest...>`:

- `drop` produces no output.
- `id` produces the input history unchanged.
- `dup` produces `<pk, pk, rest...>`. The new leftmost copy is current; later
  assignments modify only that copy.
- `filter(a)` behaves as predicate `a`.
- `h <- v` changes header `h` in the current packet and leaves every older
  packet unchanged.
- `p + q` produces the union of all histories produced by `p` or `q`.
- `p . q` runs `p`, then runs `q` on every history produced by `p`.
- `p*` runs `p` zero or more times and produces the union of the results for
  every finite iteration count, including the unchanged input history.

Outputs are sets: duplicate derivations of the same history have no observable
effect.

## Equivalence

For a declaration:

```text
check name: p == q;
```

print `name: equivalent` if and only if, for every nonempty input packet
history `hist`, the two output sets are identical:

```text
[[p]](hist) = [[q]](hist)
```

Otherwise print `name: not equivalent`.

Equivalence is history-sensitive. For example:

```text
check endpoint-is-not-enough:
  dst_port <- 443
  ==
  dup . dst_port <- 443;
```

is `not equivalent`: the current packets agree, but the right side records an
additional packet. Likewise, equality must account for the unbounded families
of histories generated by stars containing `dup`; checking executions only up
to a fixed depth is not sufficient.
